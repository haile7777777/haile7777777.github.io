es6中的解构赋值

发表于 2019-01-05 | 分类于 es6

概述

本文主要记录es6中数组，对象，字符串的结构赋值，以及一些其他相关的细节

解构赋值的概念

从数组或对象中提取值，然后对变量进行赋值，这个过程叫做解构赋值

解构赋值的本质:
本质就是”模式匹配”，只要等号左右的模式匹配，左边的变量就会被右边赋值

数组的解构赋值

数组解构的几种状态

  • 成功解构，即等号左右完全匹配 — 例如：

  • 不完全解构，即等号左右没有完全匹配,变量的值就为undefined

  • 解构失败，因为等号右边的数据结构不具备Iterator接口

默认值

ES6中严格使用”===”判断一个位置是否有值。所以，如果一个数组成员不严格等于undefined，默认值是不会生效的

对象的解构赋值

  • 对象和数组解构的不同：

    • 数组的元素按次序排列，变量的取值取决于它的位置
    • 对象的属性没有顺序，变量必须与属性同名才能取到值

  • 解决上述问题，采用别名的形式

对象解构赋值的本质：
先找到同名属性，然后再赋值给对应的变量
真正赋值的是后者（c）而不是前者（b）

  • 默认值生效的条件是，对象的属性值严格等于undefined
