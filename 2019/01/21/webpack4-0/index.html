webpack4.0

发表于 2019-01-21 | 分类于 开发工具

概述

本文主要记录最新的webpack4.0的配置使用

安装

cnpm install webpack webpack-cli -g

  • webpack4.0依赖webpack-cli,所以同时安装

简介

  • 4.0简介

    从 webpack v4.0.0 开始，可以不用引入一个配置文件。然而，webpack 仍然还是高度可配置的

    仍然还是可以对四个核心模块进行配置：

    • 入口(entry)
    • 输出(output)
    • loader
    • 插件(plugins)

  • 执行webpack命令：

    • 4.0中，会默认将src/index.js作为入口文件，输出到dist/main.js
    • 3.0中，将src/main.js作为入口文件，输出到dist/bundle.js

webpack.config.js

  • webpack.config.js的结构：就是一个导出对象，配置项遵循对象的书写格式填进去就好

  • 配置模式：mode

    • mode分为development和production

      • development：代码不压缩
      • production：代码会压缩

  • entry：指定入口文件

  • ouput：指定出口文件

  • 拓展：webpack-dev-server

    • 安装：cnpm install webpack-dev-server -D

    • package.json中配置：

      “dev”: “webpack-dev-server –open –port 3000 –contentBase src –hot”

      • –open:表示自动打开浏览器
      • –port 3000:表示默认在3000端口上执行
      • –contentBase src:表示指定打开目录
      • –hot:表示开启热更新

  • plugins: 插件目的在于解决 loader 无法实现的其他事

  • loader:

    • loader 用于对模块的源代码进行转换。loader 可以使你在 import 或”加载”模块时预处理文件
