http协议深入学习(一)

发表于 2019-01-20 | 分类于 http

概述

本系列文章对HTTP协议的学习做一个记录，主要包括http基础，特性，tcp，以及nginx配置使用等知识点

引子

  • 这里先放上一道常问的面试题作为本阶段学习的引子

    在浏览器中输入url后到http请求返回的完成流程是什么样的

  • 文字描述

    1、浏览器的地址栏输入URL并按下回车。

    2、浏览器查找当前URL是否存在缓存，并比较缓存是否过期。

    3、DNS解析URL对应的IP。

    4、根据IP建立TCP连接（三次握手）。

    5、HTTP发起请求。

    6、服务器处理请求，浏览器接收HTTP响应。

    7、渲染页面，构建DOM树。

    8、关闭TCP连接（四次挥手）

OSI模型

  • 因特网的协议栈

  • 协议栈解读
    • 应用层：
      • 是网络应用程序及他们 的应用层协议保存的地方
      • 应用层的协议包括：
        • http:80
        • SMTP:20,21
        • FTP:25
        • DNS域名解析系统
      • 应用层的协议分步在多个端系统上，信息分组叫报文
    • 运输层：
      • 运输层在应用程序端点之间传递应用层报文
      • 运输层的协议包括TCP和UDP
        • TCP:
          • 想应用程序提供面向连接的服务
          • 服务包括确保传递和流量控制
          • 将长报文划分成短报文
          • 提供拥塞控制机制
        • UDP：
          • 提供无连接的服务
          • 没有可靠性
          • 没有流量控制
          • 没有拥塞机制
      • 运输层的分组叫报文段
    • 网络层：
      • 网络层负责将数据报从一台主机移动到另一台主机上
      • 网络层的协议:IP
    • 链路层：
      • 网络层下传数据报到链路层
      • 链路层沿着路径到下个节点上传数据报到网络层
      • 链路层的数据分组叫帧
    • 物理层：
      • 物理层将链路层的帧一比特一比特的移动到下个节点
    • 表示层：
      • 表示层的作用是使通信的应用层程序能够解释交换的数据的含义
      • 服务包括：数据压缩，加密以及数据描述
    • 会话层：
      • 提供了数据交换定界和同步的功能，包括建立检查点和恢复方案的方法
